# Drom Auto Dataset Builder

Парсер и ETL-пайплайн для автоматизированного сбора, нормализации и обогащения объявлений о продаже автомобилей с сайта drom.ru.

## Назначение и бизнес-задачи
Проект формирует структурированный, чистый датасет для последующего обучения ML-моделей:
- **Для покупателей:** оценка справедливой рыночной стоимости.
- **Для продавцов:** подбор оптимальной цены размещения.
- **Для платформ:** автоматическая детекция аномальных и потенциально мошеннических объявлений.

*Архитектура проекта разделена на два репозитория: сбор данных (текущий) и ML-часть (регрессия/классификация).*

## Архитектура пайплайна
1. **Fetcher** — HTTP-клиент с retry-логикой, rate-limiting и обработкой статусов 403/429.
2. **List Parser** — парсинг страниц выдачи: извлечение `ad_id`, ссылок, превью-данных.
3. **Ads Parser** — глубокий парсинг карточек объявлений по `data-ftid` селекторам.
4. **Dataset Builder** — объединение листинга и карточек, нормализация числовых полей (цена, пробег, двигатель), маппинг городов через `towns.jsonl`.
5. **Feature Extraction** — расчет производных признаков (`age`, `price_per_hp`), текстовые метрики, риск-флаги.
6. **Pipeline Orchestrator** — многопоточный сбор, управление сессиями, атомарные чекпоинты, graceful shutdown.

## Ключевые особенности
- **Надежность:** Потокобезопасная запись, автоматический resume прерванных сессий, явная валидация типов (`dtype={"ad_id": str}` при чтении).
- **Отказоустойчивость:** Обработка сетевых ошибок, повторные попытки, лимит запросов, корректное завершение по `Ctrl+C`/`SIGTERM` с сохранением буфера.
- **Оптимизация:** Частичная загрузка, временные сессии в `sessions/`, история ID в `history_ids.txt`, корректная очистка файлов в Windows (`gc.collect()` + `sleep`).
- **Чистота данных:** Нормализация текста, удаление HTML/телефонов/ссылок, эвристический парсинг `brand/model/year` (включая multi-word марки).
- **Гибкость:** Все параметры вынесены в `config.py`, поддержка динамического именования выходных файлов.


## Конфигурация
Основные настройки находятся в src/config.py:
- TARGET_URLS: список URL для парсинга (формат: https://auto.drom.ru/{brand}/{model}/?params...)
- HEADERS: HTTP-заголовки
- DEALER_KEYWORDS, RISKY_KEYWORDS и др.: наборы слов для генерации риск-флагов
- MULTI_WORD_BRANDS: марки из нескольких слов для корректного парсинга заголовков
- Пути к директориям и файлам (data/, towns.jsonl)

Примечание: Перед первым запуском убедитесь, что файл data/external/towns.jsonl существует.

## Запуск
```bash
python src/pipeline.py
```
**Пайплайн автоматически:**
* Создаст необходимые директории.
* Пройдет по всем URL из config.py.
* Сохранит промежуточные результаты в data/raw/sessions/.
* При успешном завершении удалит временные файлы и сохранит итоговый датасет.
* Обновит data/raw/history_ids.txt для предотвращения дубликатов при повторных запусках.

## Формат выходных данных
Итоговый файл: data/processed/session_YYYYMMDD_HHMMSS_{brands}_{count}.parquet (или dataset_final.parquet по конфигу).
Содержит:
* Исходные поля объявления
* Нормализованные числовые метрики
* Расчетные фичи (age, price_per_hp)
* Текстовые флаги (is_dealer, is_reseller, has_accident, is_risky)
* Оптимизированные dtypes для снижения потребления памяти.

## Структура проекта
```drom-auto-dataset-builder/
├── data/
│   ├── external/
│   │   └── towns.jsonl          # Маппинг городов
│   ├── raw/
│   │   ├── history_ids.txt      # История обработанных ad_id
│   │   └── sessions/            # Временные чекпоинты текущей сессии
│   └── processed/               # Итоговые parquet-датасеты
├── src/
│   ├── config.py                # Конфигурация: пути, headers, URL, ключевые слова
│   ├── pipeline.py              # Точка входа и оркестрация пайплайна
│   ├── fetcher.py               # HTTP-клиент с retry и rate-limiting
│   ├── list_parser.py           # Парсер страниц выдачи
│   ├── ads_parser.py            # Парсер карточки объявления
│   ├── dataset_builder.py       # Нормализация, маппинг, merge данных
│   └── feature_extraction.py    # Генерация признаков и риск-флагов
├── requirements.txt
└── README.md
```
